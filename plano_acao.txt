Você é um desenvolvedor full-stack sênior especializado em SaaS financeiro, dashboards, análise de dados e arquitetura de sistemas.  Preciso que você implemente uma funcionalidade de dashboard financeiro no sistema atual.  O sistema já possui uma funcionalidade de transações financeiras. Portanto, o dashboard deve usar as transações já existentes no banco de dados como fonte principal de informação.  Não crie uma estrutura paralela se já existir uma model/tabela de transações. Antes de implementar, analise o projeto atual e identifique como as transações estão estruturadas.  ---  ## Objetivo da funcionalidade  Criar um dashboard financeiro mensal para que o usuário consiga acompanhar sua vida financeira com base nas transações já registradas.  O dashboard deve permitir que o usuário visualize:  - Total de entradas. - Total de saídas. - Saldo do período. - Comparação com o mês anterior. - Gastos por categoria. - Gastos por forma de pagamento. - Gastos fixos e variáveis. - Gastos recorrentes. - Reservas e investimentos. - Metas financeiras, se existirem. - Alertas financeiros. - Relatório financeiro gerado por IA.  A funcionalidade deve ser baseada em dados reais cadastrados pelo usuário.  ---  ## Antes de implementar  Analise o projeto atual e identifique:  1. Onde está a model/tabela de transações. 2. Quais campos existem na transação. 3. Como o sistema diferencia entrada e saída. 4. Como o sistema armazena categoria. 5. Como o sistema armazena forma de pagamento. 6. Como o sistema armazena usuário/dono da transação. 7. Como o sistema armazena data da transação. 8. Como o sistema armazena valor. 9. Se já existem metas financeiras. 10. Se já existem categorias de investimento/reserva. 11. Se já existe frontend, API, templates ou componentes reutilizáveis.  Não assuma nomes de tabelas ou campos sem verificar o projeto.  Caso algum campo necessário não exista, proponha a menor alteração possível.  ---  ## Escopo principal do dashboard  ### 1. Filtro de período  Criar filtro por:  - Mês. - Ano.  Opcionalmente, se já existir no sistema:  - Conta financeira. - Categoria. - Forma de pagamento.  Ao alterar o filtro, todos os dados do dashboard devem ser recalculados.  ---  ### 2. Resumo financeiro do período  Criar uma área de resumo com os seguintes indicadores:  - Total de entradas no mês. - Total de saídas no mês. - Saldo do mês. - Total reservado/investido no mês. - Total gasto no crédito. - Total gasto no débito. - Percentual da renda comprometida. - Comparação com o mês anterior.
